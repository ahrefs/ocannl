## Summary

Implement a new tutorial file `test/training/fsm_transformer.ml` that trains a small decoder-only transformer on sequences generated by a deterministic finite state machine with 8 states (2^3). The tutorial demonstrates that transformers can learn deterministic transition rules from sequence data, using existing OCANNL building blocks.

## Architecture

### Decoder-Only Model via `transformer_encoder_block`

The task calls for a **decoder-only** autoregressive transformer. OCANNL's `transformer_decoder_block` includes cross-attention (for encoder-decoder models), which we do not need. Instead, use `transformer_encoder_block` with a **causal mask** -- this is exactly a decoder-only block (masked self-attention + FFN + layer norm). This pattern is already validated in `test/operations/layer_norm_test.ml` which defines a `mini_decoder_block` using `multi_head_attention` with a mask parameter plus an FFN.

**Recommended approach**: Define a small helper (or inline) that stacks `transformer_encoder_block`-style layers but passes a causal mask. Specifically, use `multi_head_attention` directly with `~mask` plus `layer_norm` and `mlp`, following the `mini_decoder_block` pattern from `layer_norm_test.ml` lines 4-13.

### Model Hyperparameters

- **Vocabulary/states**: 8 (num_classes = 8)
- **Context window**: 16-32 tokens (sequence length for training)
- **d_model**: 32 (small embedding dimension)
- **num_heads**: 2
- **d_k, d_v**: 16 (= d_model / num_heads)
- **d_ff**: 64 (FFN hidden dimension)
- **num_layers**: 2
- **Positional encoding**: Learned (inline parameter, same pattern as `transformer` in nn_blocks.ml line 218)

## Implementation Plan

### 1. FSM Data Generator

```
(* Pure OCaml, no external data files needed *)
- Define a transition table: int array array (8 states x 2 inputs -> next state)
- Generate the table deterministically from a fixed seed
- generate_sequence: given initial state and length, produce a sequence of states
  by drawing random binary inputs at each step
- generate_batch: produce batch_size sequences of seq_len states as a flat
  Bigarray, then wrap with TDSL.rebatch
```

The FSM has 8 states and binary input at each step. The transition table maps `(state, input_bit) -> next_state`. Training sequences are just the state trajectories (the binary inputs are implicit -- the model must learn to predict the next state given the history).

### 2. Data Encoding

Follow the existing `one_hot_of_int_list` pattern from `nn_blocks.ml` (line 61):
- Input: one-hot encoded state sequences of shape `[batch; seq_len; 8]`
- Target: shifted by one position (input = tokens[0..n-1], target = tokens[1..n])
- Use `TDSL.rebatch` to create batched tensors, indexed by `batch_n` static symbol

### 3. Model Construction

```
(* Pseudocode for the model *)
let decoder_only_block ~label ~num_heads ~d_k ~d_v ~d_ff () =
  let mha = multi_head_attention ~label ~num_heads ~d_k ~d_v () in
  let ffn = mlp ~label ~hid_dims:[d_ff] () in
  let ln1 = layer_norm ~label () in
  let ln2 = layer_norm ~label () in
  fun ~train_step ~mask x ->
    let x1 = ln1 (x + mha ~train_step ~mask x) in
    ln2 (x1 + ffn x1)

(* Stack num_layers blocks *)
let model = ... fun ~train_step ~mask input ->
  let embedded = {embed; o=[d_model]} * input + {pos_encoding} in
  let hidden = List.fold layers ... in
  {w_out} * hidden
```

### 4. Causal Mask

Use the exact pattern from `attention_test.ml` / `layer_norm_test.ml`:

```
let mask =
  NTDSL.init ~l:"mask" ~prec:Ir.Ops.single ~b:[seq_len] ~i:[seq_len] ~o:[]
    ~f:(function
      | [| s; t |] -> if s >= t then 1. else 0.
      | _ -> failwith "unexpected indices")
    ()
```

### 5. Loss Computation

Cross-entropy loss, following the pattern from `bigram_mlp.ml`:
- Compute softmax over the 8 output classes
- Multiply by one-hot targets, sum, negate, log
- Average over batch

### 6. Training Loop

Follow `bigram_mlp.ml` structure:
- `IDX.get_static_symbol` for batch index and step counter
- `Train.grad_update` + `Train.sgd_update` with decaying learning rate
- `Train.init_params`, `Train.to_routine`, `Train.run`
- Print epoch loss; optionally track per-epoch accuracy
- ~50-100 epochs should suffice for convergence on this simple problem

### 7. Evaluation

After training:
- Generate fresh test sequences from the FSM
- Run forward pass, check argmax predictions against ground truth
- Print accuracy (expect near 100% for a well-trained model on a deterministic FSM)
- Optionally print the learned transition table vs ground truth

## Key Risks and Mitigations

1. **Shape inference with sequences**: OCANNL's shape inference must handle the `[batch; seq_len; d_model]` tensor flow through attention. This is already tested in `attention_test.ml` and `layer_norm_test.ml`, so should work.

2. **No tensor slicing for teacher forcing**: OCANNL lacks easy tensor slicing. Pre-shift the data at generation time: `input_seqs = states[0..n-1]`, `target_seqs = states[1..n]`, as separate tensors. This avoids runtime slicing.

3. **Convergence**: A deterministic FSM with only 8 states and short sequences should be trivially learnable. If the model doesn't converge, reduce model size or increase training data.
