# OCANNL v1.1 Freshness Sweep — Triage Proposal

## Goal

Resolve the v1.1 milestone backlog drift detected on 2026-05-01: most
2024-vintage v1.1 explore/enhancement tasks have premises that no longer
match the codebase or the user's strategic direction. The sweep produces
this triage proposal as a one-shot artifact; the user reviews each row and
gives per-row feedback ("keep / close / defer / promote"), and the
implementation-phase coder executes the agreed-upon dispositions via
`ludics tasks` CLI commands plus a closing ROADMAP.md edit.

This memo is the **load-bearing artifact**. Once it lands and the user
batch-approves per-row dispositions, execution is a CLI script of ~5–10
lines plus a ROADMAP.md edit. No new infrastructure required. Per task
ACs, the proposal review IS the user-confirmation gate (one round-trip),
not N round-trips via dashboard tiles.

Linked harness task: `task-a993fe29`. Driving observation: 5/7 of the
already-audited 2024-vintage tasks resolved to close-or-defer. Multiple
v1.1 issues lost their causal premises when multi-streaming was removed
in 0.6.1 (gh-ocannl-341), or were retracted by the user themselves in
historical issue comments (e.g. #123 in 2025-09-23, #137 in 2023-09-27).

- [ ] **AC-2. Per-row evidence.** Every row's `evidence` cell cites at
  least one of: (a) a specific repo-root-relative doc/memo path that
  exists on disk in the project worktree; (b) a dated user-historical
  retraction/deprioritization comment, with the date in the cell so
  the falsifier can scan for it; (c) a current codebase pointer by
  `file:line` or `file:symbol`; or (d) a GitHub-issue-body pointer
  (target issue number + `createdAt` date or referenced-issue
  `milestone`/`state`) verifiable by `gh issue view <n>` — added in
  round 3 so rows whose evidence is GitHub-side rather than
  repo-local (e.g. #297 cites the issue body's ModernBERT/Answer.AI
  source URLs and the v0.7.1 prerequisite `#377` milestone) cannot
  hide behind vague prose.

  Falsifier (three sub-checks; pass = all three empty):

  ```bash
  # 2a — repo-local doc/memo paths must resolve on disk
  cd ~/ocannl-staging && \
    grep -oE 'docs/[a-z0-9/_-]+\.md' docs/proposals/v1.1-freshness-sweep.md \
    | sort -u \
    | while read p; do test -f "$p" || echo "MISSING-PATH: $p"; done

  # 2b — every row carries at least one qualifying evidence anchor.
  # A row qualifies if its evidence cell contains one of:
  #   - a `docs/.../*.md` path (sub-check 2a covers existence),
  #   - a date matching YYYY-MM-DD (covers (b) retraction comments and
  #     (d) GitHub-issue createdAt cites),
  #   - a `file.ml:NNN` or `file.ml` codebase pointer,
  #   - a `#NNN` issue cross-reference that points to a separate issue
  #     (i.e. not just the row's own number)
  cd ~/ocannl-staging && \
    awk -F'|' '
      /^\| \[#[0-9]+\]/ {
        match($2, "[0-9]+"); row_n = substr($2, RSTART, RLENGTH);
        cell = $4;
        has_path = match(cell, "docs/[a-z0-9_/-]+\\.md") > 0;
        has_date = match(cell, "[0-9][0-9][0-9][0-9]-[0-9][0-9]-[0-9][0-9]") > 0;
        has_code = match(cell, "[a-z_]+\\.ml(:[0-9]+|:[A-Za-z_]+)?") > 0;
        has_xref = 0;
        n = split(cell, refs, "#");
        for (i = 2; i <= n; i++) {
          ref_n = refs[i]; gsub(/[^0-9].*/, "", ref_n);
          if (ref_n != "" && ref_n != row_n) { has_xref = 1; break; }
        }
        if (!(has_path || has_date || has_code || has_xref))
          print "MISSING-EVIDENCE: row #" row_n;
      }' docs/proposals/v1.1-freshness-sweep.md

  # 2c — every GitHub-issue cross-reference in an evidence cell
  # corresponds to a real issue (catches typos / dead refs)
  cd ~/ocannl-staging && \
    awk -F'|' '
      /^\| \[#[0-9]+\]/ {
        match($2, "[0-9]+"); row_n = substr($2, RSTART, RLENGTH);
        n = split($4, refs, "#");
        for (i = 2; i <= n; i++) {
          ref_n = refs[i]; gsub(/[^0-9].*/, "", ref_n);
          if (ref_n != "" && ref_n != row_n) print ref_n;
        }
      }' docs/proposals/v1.1-freshness-sweep.md \
    | sort -u \
    | while read n; do \
        s=$(gh issue view "$n" --repo ahrefs/ocannl --json state -q .state 2>/dev/null); \
        [ -z "$s" ] && echo "MISSING-XREF: #$n"; \
      done
  ```

  The 2b check is the load-bearing one for the reviewer's round-2
  concern: a row with no `docs/.../*.md` path, no `YYYY-MM-DD` date,

  different issue) prints `MISSING-EVIDENCE:` and fails AC-2 — the
  prior `2a`-only falsifier would have silently passed such a row.
